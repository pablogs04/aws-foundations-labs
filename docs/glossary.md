# glossary

## AWS and DevOps glossary

- **account**: the AWS account boundary where resources and billing live
- **root user**: the highest-privilege identity for account-level tasks only
- **IAM user**: a long-term identity for a person or workload
- **MFA**: multi-factor authentication for stronger login security
- **least privilege**: granting only the permissions required to do the job
- **named profile**: a local AWS CLI profile with an explicit name
- **budget alert**: a spending notification, not an automatic stop
- **cleanup**: removing or stopping resources after validation
- **blast radius**: the scope of potential damage from a mistake
- **validation**: the checks that prove a task worked correctly
- **rollback**: the path to revert a change safely
- **tracking branch**: the local branch linked to a remote branch in Gits
